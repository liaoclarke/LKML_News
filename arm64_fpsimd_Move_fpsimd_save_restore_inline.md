# arm64: fpsimd: Move fpsimd save/restore inline

---

## 更新 - 2026-05-21 14:25 UTC

## 核心话题
本邮件讨论的是 ARM64 架构中 FPSIMD（浮点和单指令多数据）寄存器保存/恢复逻辑的实现方式重构。当前内核与 KVM 的 hyp 模式分别维护了独立的、用汇编宏编写的保存/恢复序列，虽然共享宏代码，但仍存在代码重复和细微差异，且宏接口使用数字索引来指定寄存器（例如 `fpsimd_save x0, 8` 表示使用 x0 和 x8 寄存器），可读性差、易出错。此外，SVE 和 FPSIMD 对 FPSR/FPCR 的保存/恢复逻辑各自独立，未能复用，而汇编形式的序列无法被 KASAN 等工具插桩检测，增加内存安全问题排查难度。

为了改善上述情况，补丁将 FPSIMD 寄存器保存/恢复从外部汇编文件移到 C 代码中的内联汇编里，同时让 FPSR+FPCR 的保存/恢复与 SVE 共享同一段逻辑。这样消除了多处维护的重复代码，统一了接口，并能够通过编译器对内存参数进行插桩检测（nVHE hyp 代码由 Kbuild 隐式禁用插桩）。补丁还处理了 FPCR 恢复方式的历史分歧：2014 年的 FPSIMD 序列曾使用条件写入 FPCR 以优化性能（commit `5959e25729a5`），但 2017 年的 SVE 序列并未沿用这一优化（commit `1fc5dce78ad1`），补丁作者选择采用更简单的无条件写入方式，并认为该差异在实际中并不重要。
