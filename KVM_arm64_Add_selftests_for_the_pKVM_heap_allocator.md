# KVM: arm64: Add selftests for the pKVM heap allocator

---

## 更新 - 2026-05-20 16:26 UTC

## 核心话题
该邮件是 Linux 内核邮件列表中 Vincent Donnefort 提交的 pKVM 系列补丁的第十部分，主题为“为 pKVM 堆分配器添加自测试”。pKVM（保护型 KVM）在 ARM64 架构下将 hypervisor 运行在 EL2 异常级别，其堆分配器是 Hypervisor 内部用于动态内存管理的关键组件。为了确保堆分配器在 EL2 受限环境下的正确性与鲁棒性，补丁引入了一套完整的运行时自测试（selftests），该测试在 `CONFIG_NVHE_EL2_DEBUG` 配置选项开启时，于 pKVM 初始化阶段在 EL2 内部执行。

邮件中明确列出了自测试所覆盖的核心机制：
- 对超大尺寸分配的处理，验证分配器在请求超出当前可用空间时的行为；
- 基本分配与对齐保证，测试常规 size 的分配是否能按预期返回正确对齐的内存块；
- 块的回收（recycling）、分割（splitting）与合并（merging）逻辑，这些是堆分配器减少碎片、提高空间利用率的关键操作；
- 内存回收（reclaim）机制，验证 EL2 堆在内存压力下能否向主机申请回收已释放的页；
- 内存补充（topup）流程，确保当堆内内存不足时，能够通过定义的接口向主机请求额外交付物理页。

从修改的代码片段可见，该自测试通过新增的 SMC 调用接口（`__KVM_HOST_SMCCC_FUNC___pkvm_hyp_alloc_selftest`）与配套的请求类型（`PKVM_HYP_REQ_HYP_ALLOC_SELFTEST`、`PKVM_TOPUP_HYP_ALLOC_SELFTEST`）实现。这改变了 pKVM 与主机之间的交互协议，使得在调试模式下主机能够向 EL2 发起自测试请求或为自测试提供额外内存资源。补丁中修改的文件包括 `asm/kvm_asm.h`、`asm/kvm_pkvm.h` 以及 hyp 内部分配器头文件 `nvhe/alloc.h`，将自测试所需的枚举和函数编号加入原有框架，从而在不影响正常运行时路径的前提下嵌入验证逻辑。

技术动机在于 pKVM 堆分配器直接运行于 EL2，其正确性对虚拟化安全与稳定性至关重要，但传统测试方法难以直接触及 EL2 执行环境。通过将自测试集成到初始化流程中并由内核配置控制，开发者可以在开发与调试阶段自动验证分配器的核心功能，及时捕获边界条件错误或回归问题。该补丁作为系列中的一环，体现了将测试与功能代码同步合入的开发实践，有利于提高 pKVM Hypervisor 整体软件质量。

## 下一步改进方向
1. 社区审查：需要 ARM64 及 KVM 子系统维护者对自测试的实现方式（例如是否应将测试嵌入初始化流程而非作为独立模块）、SMC 接口的扩展是否合理、测试覆盖是否充分等进行审核；
2. 可能的设计调整：根据反馈，自测试的触发机制或内存 topup 实现可能需要修改，例如改为仅在显式触发时运行而非每次 init 都执行，以降低生产内核的代码风险；
3. 测试覆盖率扩展：可考虑增加更多场景，如并发分配、随机化测试或内存压力模拟，以进一步强化验证能力；
4. 补丁重发：若评审中有修改要求，需要发布 v2 版本并在整个系列中同步更新相关接口。
