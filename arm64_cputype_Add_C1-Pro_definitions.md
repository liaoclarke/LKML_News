# arm64: cputype: Add C1-Pro definitions

---

## 更新 - 2026-05-21 09:50 UTC

## 核心话题
本邮件线程讨论的是 Linux 稳定版维护者 Sasha Levin 通过 AUTOSEL 机制自动选择到 5.10.y 稳定分支的一个补丁——“arm64: cputype: Add C1-Pro definitions”（上游提交 2c99561016c591f4c3d5ad7d22a61b8726e79735）。该补丁为 ARM64 架构的 C1‑Pro 处理器添加了 cputype 宏定义（如 CPU 部件号、变体等），原本是为后续的勘误表（errata）检测补丁做准备。

Mark Rutland 指出，这个定义仅仅被一个只适用于 v6.18 及以上主线版本（以及下游 android16‑6.12 树）的 workaround 所需，并无任何其他在 5.10 内核中依赖它的代码。因此他认为，除非存在实际依赖，否则完全没有必要将此补丁回传到 v5.10.y 长期支持分支。这一判断基于稳定版内核的“不要引入不需要的代码”原则：只有真正修复用户可见问题或为后续已回传的修复提供依赖的补丁才应进入稳定树，否则只会增加维护负担和引入不必要的代码变动。

Sasha Levin 在读到 Mark 的解释后迅速表示同意，并承诺会丢弃该补丁，不再将其合并入 5.10.y 分支。整个讨论简洁而直接，体现了稳定分支维护过程中对补丁必要性进行二次确认的典型流程。

已达成明确共识：**该补丁不需要回传到 v5.10.y 稳定分支**。Mark 指出了补丁仅服务于一个尚未在 5.10 中存在的 workaround，Sasha 接受了这一理由并决定丢弃该补丁。无遗留分歧。
