## 核心话题
本邮件线程围绕 ARM64 pKVM（protected KVM）初始化过程中 `__pkvm_init_vm` 函数的错误路径修复展开。Vincent Donnefort 发布了一个补丁系列系列（v3），旨在修复 Sashiko 报告的一个潜在引用计数泄漏问题：在 `insert_vm_table_entry` 调用失败的罕见情况下，页面引用计数未能正确释放。此外，补丁还在 `__pkvm_hyp_donate_host` 中添加了故障安全机制，确保该函数不会泄漏已被引用计数的页面。v3 版本与前几个版本相比的关键改动是强制外部页面必须为 order-0，从而使 `vmemmap` 中的 `->order` 字段完全成为 hyp pool 的私有实现细节，避免外部页面携带非零 order 导致引用计数管理的混乱。
