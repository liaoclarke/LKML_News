# arm64+KVM: FPSIMD/SVE/SME cleanups

---

## 更新 - 2026-05-21 14:25 UTC

## 核心话题
本邮件是 Mark Rutland 提交的针对 ARM64 架构下 FPSIMD/SVE/SME 状态管理代码的清理系列补丁（PATCH 00/18）。系列补丁纯粹是代码清理重构，不涉及任何 Bug 修复。核心目标是为后续功能扩展（例如为 KVM 增加 SME 支持）铺平道路，同时增强代码的可维护性和可调试性。

技术动机与关键要点如下：
1. **内存状态管理的加固**：将 SVE 和 SME 的内存状态封装为不透明类型 `struct sve_state` 和 `struct sme_state`，避免直接操作内部字段导致的误用。同时简化 SVE 状态的保存/恢复调用约定，原来需要传递指向状态内部 FFR 的指针，现在改为直接传递状态基地址指针，消除了这个笨拙的设计。
2. **消除代码重复**：将 FPSIMD/SVE/SME 操作例程迁移到内联汇编中，使得 KVM 和内核其他部分可以共用同一套辅助函数，无需再包装汇编宏。
3. **提升可读性与简洁性**：尽量减少汇编序列，将地址生成和控制流改用更清晰的 C 代码实现；移除不再必要的、晦涩的汇编宏；利用汇编器对 SVE 和 SME 的直接支持，大幅简化代码。
4. **增强调试能力**：在保存/恢复函数中显式加入 KASAN 和 KCSAN 的检测插桩，使内存安全问题和并发问题能被动态检测工具捕获。对于 nVHE hyp 对象，已抑制该插桩，确保不影响 KVM 的运行。作者通过查看编译时标志及反汇编确认了这一点。
5. **移除不必要的代码**：借助现代汇编器特性，去掉复杂的汇编宏，使代码更加精简易读。

提交者已在一系列编译器版本（GCC 8.1.0 至 15.2.0、LLVM 15.0.7 及 21.1.8，搭配不同 binutils）下完成编译测试，并在具备 SVE 和 SME 能力的模型上进行了启动测试（邮件被截断，可能提及 KASAN 等测试场景）。
